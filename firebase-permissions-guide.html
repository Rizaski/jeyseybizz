Firebase Permissions Guide

Fix "Missing or insufficient permissions" error

‚ùå The Problem

Error Message:

"Missing or insufficient permissions"

What it means:

Your Firestore security rules are blocking read/write access to the database.

Impact:

Orders cannot be saved to or loaded from Firebase.

‚úÖ The Solution

Quick Fix:

Update your Firestore security rules to allow read/write access.

Fallback:

The admin panel will automatically use local storage when Firebase is unavailable.

üîß Step-by-Step Fix

Step 1: Go to Firebase Console

Visit https://console.firebase.google.com

Select your project: otomono-c9938

Step 2: Navigate to Firestore

In the left sidebar, click on "Firestore Database"

Then click on the "Rules" tab

Step 3: Update Security Rules

Replace the existing rules with the following:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Allow read/write access to orders collection
    match /orders/{document} {
      allow read, write: if true;
    }
    
    // Allow read/write access to users collection
    match /users/{document} {
      allow read, write: if true;
    }
    
    // Allow read/write access to systemSettings collection
    match /systemSettings/{document} {
      allow read, write: if true;
    }
  }
}

Step 4: Publish Rules

Click the "Publish" button to save the changes

Wait for the rules to be deployed (usually takes a few seconds)

Step 5: Test the Fix

Go back to your admin panel and try:

  • Creating a new order
  • Loading existing orders
  • Editing order status
